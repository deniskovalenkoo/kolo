# Claude rules for the kolo.xyz Astro project

## Stack
- **Astro 5** (static SSG)
- **TypeScript strict**
- **Vercel** deployment (push to `main` = auto-deploy)
- **Sanity** (planned, for blog only)
- **Notion** (planned, only if we add forms)

## Languages
`en` (default, no URL prefix), `ua` (`/ua/...`), `ru` (`/ru/...`)

## Sources of truth
- **Pages currently live on kolo.xyz**: scraped 1:1 by `scripts/scrape-webflow.mjs`.
  The HTML is stored in `src/_scraped/` and rendered through `src/layouts/WebflowPage.astro`.
  **Do not hand-edit `src/_scraped/` files** — they are regenerated. Edit the live
  Webflow page first, then re-scrape; OR start migrating that page off Webflow into
  hand-written Astro components.
- **New pages** (programmatic SEO, etc.): hand-written Astro components, not scraped.

## File-path conventions
Astro maps `src/pages/x.astro` → URL `/x`. Mirror kolo.xyz URLs exactly:
- `/buy` → `src/pages/buy.astro`
- `/ru/buy` → `src/pages/ru/buy.astro`
- `/quiz/start` → `src/pages/quiz/start.astro`

When `git add`-ing files with brackets in zsh, quote the path:
```bash
git add 'src/pages/[lang]/foo.astro'  # ✅
```

## Asset hosting
Currently all images/CSS/JS load from `cdn.prod.website-files.com` (Webflow's CDN)
because the scraped HTML preserves original URLs. Local copies live in `public/cdn/`
for the day we decide to self-host.

Don't add new CSS classes — match the existing Webflow class names (or use inline
`style="..."` as a last resort) until we re-build pages from scratch.

## What NOT to touch
- `public/cdn/` — auto-downloaded by scraper
- `src/_scraped/` — auto-generated by scraper
- `src/pages/<scraped-page>.astro` — auto-generated by scraper

## Adding a new page (not from Webflow)
1. Create `src/pages/your-path.astro`
2. Don't import `WebflowPage` (that's only for scraped pages)
3. Use a fresh layout (build one if needed) so the page can be styled freely

## Build & test
```bash
npm run dev          # local preview at http://localhost:4321
npm run build        # build to /dist; CI runs this on every push
```

## When the user asks to change a scraped page
1. Locate the corresponding `src/_scraped/<file>.html`
2. Either edit the HTML directly **and** add a note that the source diverged from Webflow,
   OR re-build the page from scratch as a hand-written Astro component (preferred long-term).
